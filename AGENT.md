# AGENT.md — 每天一个小好奇 内容生成指南

## 项目背景

这是一个面向 11 岁儿童的每日好奇心平台。设计极简，内容精炼，每篇包含一个有趣的科学或知识小发现，通常配一张信息图。

**目标读者：** 11 岁儿童（2024年出生，内容随孩子成长调整）
**核心原则：** 简短、有趣、让人"哇"一下

**当前约定（图片）：** 暂不生成信息图——不要调用 `image_generate`、不要运行 `generate-images.js`。新内容的 `image` frontmatter 仍按规范填写预期路径，便于日后统一补图。

### 技术栈

- **Eleventy 3.0** — 静态站点生成器
- **Nunjucks (.njk)** — 模板引擎
- **markdown-it + gray-matter** — Markdown 处理与 frontmatter
- **构建：** `npm run build` | **开发：** `npm start`

## 内容结构

每篇内容是一个 markdown 文件，放在 `src/content/YYYY-MM/` 目录下。

### 文件命名
```
YYYY-MM-DD.md
例如：2026-01-15.md
```

### 目录结构
```
项目根目录/
├── generate-images.js   # 批量补图脚本（当前暂停使用；需 MINIMAX_API_KEY）
└── src/
    ├── content/         # 每日内容 Markdown
    │   ├── 2026-01/
    │   ├── 2026-02/
    │   ├── 2026-03/
    │   └── 2026-04/
    ├── images/          # 信息图（与正文日期对应）
    ├── _includes/       # Nunjucks 模板组件
    └── index.njk        # 主页模板
```

### Frontmatter 格式
```yaml
---
date: 2026-01-15
title: 为什么天空是蓝色的？
subtitle: 光与大气层的奇妙游戏
image: ../../images/2026-01/2026-01-15.jpg
---
```

**注意：** image 路径为相对路径 `../../images/YYYY-MM/YYYY-MM-DD.jpg`，不要用外部 URL。

### 标题与副标题
- **标题（`title`）要足够引人好奇**：读者一眼就想点进去或读完——优先用**问题**、**反常识**、**具体画面**或**小悬念**，避免像百科词条那样平淡（例如少用单独的名词短语当标题，如「光的散射」）。
- 好标题通常满足至少一条：让人心里冒出「真的吗？」「为什么会这样？」「我从没想过」；若标题本身较平，正文开头必须立刻把悬念拉满。
- **副标题（`subtitle`）**：补充角度、语气或比喻，和标题形成「钩子 + 一点氛围」，不要与标题简单重复。

### 正文结构
- 开头用一个有趣的问题或事实吸引注意
- 2-3 段解释，每段 2-3 句话
- 用 **加粗** 高亮关键词
- 总字数：150-250 字（中文）
- 结尾可以留一个小问题引发思考

### 示例正文

```
天空为什么是蓝色的？这不是一个简单的问题，答案藏在阳光和空气的对话里。

阳光看起来是白色的，但它其实是由 **红、橙、黄、绿、蓝、靛、紫** 七种颜色混合而成的。其中，蓝光波长较短，容易被空气中的微小颗粒散射向四面八方，所以我们不管朝哪个方向看，都能接收到被散射的蓝光。这就是为什么整个天空都泛着蓝色。

但为什么不是紫色的呢？虽然紫光的波长比蓝光更短，散射更强，但阳光中紫光的含量本来就少，而且我们的眼睛对蓝光更敏感。所以最终是蓝色赢得了这场颜色战争。

有趣的是，在日出和日落时，阳光需要穿过更厚的大气层，蓝光被散射殆尽，只剩下红光和橙光，所以我们会看到绚丽的朝霞和晚霞。🌅
```

## 内容主题指南

### 科技类（优先）
- **物理**：光、声、力、能量、电、磁、量子趣闻
- **天文**：太阳系、星星、黑洞、宇航、UFO与外星人
- **生物**：动物行为、植物秘密、人体奥秘、进化、基因为什么有趣
- **地球科学**：板块构造、天气、海洋、火山、矿物宝石
- **科技**：编程逻辑、人工智能、机器人、材料科学、互联网趣闻

### 人文历史类（中华文明为主）
- **中国古代科学**：四大发明、古代数学（如祖冲之圆周率）、天文学（如张衡地动仪）、医学（如华佗、李时珍）
- **历史事件**：丝绸之路、茶文化、瓷器故事、古代外交趣闻
- **文字与语言**：汉字演变、成语故事、多音字趣解、古代称呼揭秘
- **地理与城市**：大运河、都江堰、敦煌莫高窟、各地地名由来
- **历史人物**：帝王轶事、诗人趣闻、古代工匠的发明
- **西方人文**：古希腊罗马、科学革命、文艺复兴、著名博物馆

### 穿插主题
- 数学的有趣现象（不要纯计算，要故事性）
- 地理冷知识（世界各地奇怪事实）
- 简单经济或商业趣闻（如：冰淇淋为什么有桶装？）

### 内容特点
- **具体数字**：给出真实数据比模糊描述更有冲击力（"1000万只" 比 "很多" 好）
- **对比**：用熟悉的东西做对比（"比操场还大" "比眨眼还快"）
- **提问**：每篇至少一个问题，让孩子思考
- **惊喜感**：结尾要有"原来如此"的感觉

### 信息图 Prompt 模板
```markdown
[简洁英文描述画面的主体内容，风格为简约科普插画，有视觉中心，避免文字]

例如：
- 光谱分散：白色的光通过三棱镜分散成七色彩虹，简洁科学插画风格
- 蓝色天空：仰望晴朗天空，蓝渐变到白的纯净画面，儿童科普风格
- DNA双螺旋：微观世界的旋转楼梯，蓝色紫色为主，科学又梦幻
```

### 图片要求
- 风格：简约科普插画，儿童友好
- 比例：16:10 或 1:1
- 不要文字（图片中不要有文字）
- 颜色明快但不刺眼

### 生成方式
- **单篇 / 交互**：使用 `image_generate`（或等价能力）生成后保存到 `src/images/YYYY-MM/YYYY-MM-DD.jpg`。
- **批量补图**：仓库根目录 `generate-images.js`，需设置环境变量 `MINIMAX_API_KEY`，运行 `node generate-images.js`（脚本内维护日期与 prompt 列表）。

## 生成流程

当被要求生成某日期的内容时：

1. **确定日期范围**：一次生成一个月的量（1-10篇）
2. **选主题**：从上面的主题列表中选择当天适合的内容
3. **撰写正文**：按上面的结构写，控制字数
4. **图片**：暂不生成；frontmatter 里 `image` 仍写 `../../images/YYYY-MM/YYYY-MM-DD.jpg` 规范路径
5. **保存文件**：`src/content/YYYY-MM/YYYY-MM-DD.md`
6. **构建验证**：`npm run build` 确保 Eleventy 构建无错误
7. **版本控制**：仅在用户明确要求时再 `git commit` / 推送；否则只完成本地文件与构建验证。
